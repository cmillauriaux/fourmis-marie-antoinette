Le projet

Pourquoi ce site ? Pour plusieurs raisons en fait : j'aime découvrir de nouvelles choses, faire des expériences, observer, apprendre, et bidouiller chez moi. Et c'est pour cette raison que le Père Noël (non en fait ma conjointe qui m'encourage dans toutes mes lubies !) a déposé sous le sapin une fourmilière. C'est donc l'occasion pour moi de commencer ce nouveau projet, qui au delà de me faire découvrir le mode de vie fascinant de ces insectes sociaux, va me permettre d'expérimenter divers technologies (Angular 2, Ionic 2, Go, Google Cloud) et de les mettre en place dans le but de suivre et de comprendre l'évolution de ma colonie de fourmis.

Ce site a donc comme objectif de faire découvrir à tous les curieux l'élevage de fourmis, de leur permettre de comparer minute après minute, heure après heure, semaine après semaine l'évolution de la fourmilière et à tous les développeurs de jeter un oeil aux technologies qui m'ont permis de créer ce laboratoire en ligne. Amis curieux, amis développeurs, amis tout court, soulevez avec moi le capôt de ce fabuleux projet !

Zoom sur les technos

Serveur

  • Langage : Go
  • Hébergement : Google AppEngine sur Google Cloud
  • Base de données : Google DataStore
  • Stockage des fichiers : Google Storage
  • Architecture : API REST avec une couche de persistance pour les requêtes en BDD

Pourquoi ce choix ? Parceque que Go est mon langage fétiche pour développer des serveurs d'applications : typage fort, compilation, rigueur de la syntaxe, simplicité d'utilisation et nombreuses librairies tierces disponibles. C'est aussi l'occasion de tester Google Cloud qui propose une multitude de services dans le cloud pour construire une application sans se soucier de l'aspect système. Le petit plus ? Scaling automatique de l'infrastructure en fonction de la charge !

Prise de vue

  • Langage : Go + bash
  • Plateforme : Raspberry Pi 3
  • Système d'exploitation : Raspbian
  • Matériel de prise de vue : Raspberry Cam
  • Architecture : Une tâche cron lance la capture et l'envoi vers Google Storage toutes les minutes, via un script bash et un petit programme écrit en Go

Pourquoi ce choix ? Parcequ'un Raspberry ca ne coûte pas grand chose à l'achat, ca ne consomme rien même en restant allumé en permancence et ça permet d'installer un système Linux !

Le site

  • Langage : Angular 2
  • Framework visuel : Bootstrap 3
  • Système d'authentification : OAuth2 pour l'identificat des administrateurs
  • Outils : ng-cli pour assister le développeur durant toutes les phases du projet
  • Architecture : L'application Angular 2 s'alimente aurpès des services REST exposées par le serveur et affiche ensuite les images en piochant directement dans Google Storage

Pourquoi ce choix ? Parceque je suis un grand fan d'AngularJS pour développer des applications web dynamiques rapidement et que c'est l'occasion de mettre en pratique mes connaissances fraichement acquises sur Angular 2.
